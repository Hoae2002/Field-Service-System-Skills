# Field Service System Style

Use this skill to keep「小精邻到家」用户端移动端 App / 小程序界面统一、清爽、轻量、亲和、可信赖。

## Workflow

1. Identify the requested page type, such as city selection, city search, city voting, voting confirmation, voting success, empty state, service list, or appointment flow.
2. Design only for mobile App / mini-program screens unless the user explicitly asks for another platform.
3. Apply the visual foundations below before adding page-specific details: `375×812px`, `#F7F7F7` page background, white cards, green brand accents, compact typography, 8px cards, capsule primary buttons, and normal line icons.
4. Fit each designed or developed screen into the target viewport without visible scrollbars; compress spacing, split dense content into tabs/pages, or use fixed internal panels instead of allowing page-level overflow.
5. Keep the result editable and componentized for Figma: navigation bar, search box, cards, list items, tags, buttons, bottom panel, dialog, and empty-state illustration should be separable.
6. Reject visual directions that conflict with this system, such as desktop dashboards, dark technology themes, emoji icons, heavy shadows, large glassmorphism, single-character icon boxes, page-level scrollbars, or unstructured free-floating UI.
7. When writing AI image or UI-generation prompts, include the constraints from section 9 and adapt them to the specific page.

# 用户端 UI 视觉设计规范｜小精邻到家

> 文件用途：用于后续投放到文生图 AI / UI 生成 AI 中，作为统一的用户端移动端界面设计方向、颜色规范、组件风格、页面结构和生成约束依据。  
> 适用范围：用户端移动端 App / 小程序界面。  
> 当前参考页面：城市选择、城市搜索、城市投票、投票确认、投票成功等移动端页面。  
> 生成目标：确保 AI 生成页面在视觉风格、颜色、组件、图标、排版和自动布局逻辑上保持统一。

---

## 1. 总体设计方向

本项目为「小精邻到家」用户端移动端 UI，整体视觉应体现：清爽、轻量、亲和、可信赖、服务型、本地生活、到家服务、移动端操作友好。

界面不走厚重商业后台风，也不走强科技大屏风，而是偏向轻量化 C 端 App / 小程序风格：白色卡片、浅灰背景、绿色品牌强调、圆角组件、简洁列表、低干扰信息层级。

### 1.1 核心关键词

- 清爽干净
- 绿色到家服务感
- C 端移动应用
- 轻量卡片化
- 低饱和背景
- 柔和圆角
- 明确信息层级
- 少阴影或无阴影
- 适合小程序 / App 页面
- 适合本地生活、上门服务、城市服务、预约服务类产品

### 1.2 视觉基调

整体界面应以浅色为主，大面积使用 `#F7F7F7` 作为页面默认背景，模块区域使用白色卡片承载内容，品牌绿色用于主要操作、当前状态、重点数字、标签和引导按钮。

页面需要有轻微的品牌氛围，但不要过度装饰。可以在页面顶部使用浅绿色渐变氛围背景，例如从浅薄荷绿过渡到浅灰背景，形成柔和、自然、不压迫的视觉效果。

### 1.3 推荐视觉感受

- 背景轻、内容清晰、操作入口突出。
- 绿色只用于关键动作和重点信息，不要全页面泛滥使用。
- 信息模块之间要有足够留白，避免长列表和功能区过度拥挤。
- 卡片边界主要依赖背景差异和圆角，不依赖重阴影。
- 线条和分割线尽量淡，使用低透明黑色或浅灰色。

### 2.2 字体规范

优先使用中文无衬线字体，整体字体应干净、现代、易读。

#### 推荐字体

- 中文主字体：`Source Han Sans CN / 思源黑体`
- iOS 小程序常用字体：`PingFang SC / 苹方`
- 导航标题：`Noto Sans SC Medium` 或 `PingFang SC Medium`
- 数据数字：`DINPro Bold`
- 强营销型成功标题可使用更有力量的黑体，例如 `HYFengShangHei 95W`，但只建议用于少量重点标题。

#### 字号层级

| 场景 | 字号 | 字重 | 颜色 |
|---|---:|---|---|
| 顶部导航标题 | 15–16px | Medium | `#000000` |
| 页面模块标题 | 16px | Medium | `#333333` |
| 列表正文 | 14px | Regular | `#333333` |
| 说明正文 | 13px | Regular | `#777777` |
| 操作文字 | 12px | Medium | `#1DC2A5` |
| 占位提示 | 12px | Regular | `#999999` |
| 标签文字 | 11px | Medium | `#1DC2A5` |
| 主按钮文字 | 14px | Bold / Medium | `#FFFFFF` |
| 重点数字 | 36px | Bold / DINPro | 渐变 `#24D2A3 → #22BEA3` |

#### 字体使用原则

- 文字尽量少而清晰，不做大段堆叠。
- 标题与正文之间保持明确层级，标题不宜过大。
- 移动端列表页面以 14px 正文为主，说明文字用 12–13px。
- 关键操作文字使用绿色，但不要把普通正文全部做成绿色。
- 数字类信息可使用 DINPro，增强数据感和视觉重点。

## 3. 页面尺寸与布局规范

### 3.1 画板尺寸

生成移动端页面时，建议统一使用：

```text
375 × 812 px
竖版手机界面
iPhone / 小程序页面比例
```

禁止生成横版、平板尺寸、网页大屏尺寸。页面必须是窄屏竖向结构。

开发或生成页面时，默认内容必须完整适配 `375×812px` 可视区域，不要出现页面级滚动条。内容过多时优先减少视觉噪音、收紧间距、分组折叠、分页或拆成多个页面，而不是让整体页面纵向滚动。

### 3.2 页面边距

| 场景 | 数值 |
|---|---:|
| 页面左右安全边距 | 12px |
| 卡片内部横向 padding | 12–20px |
| 搜索栏外边距 | 左右 10px |
| 主按钮左右边距 | 20px |
| 底部面板内容边距 | 20px |
| 模块间距 | 10–14px |
| 列表项间距 | 12px 左右 |

### 3.3 页面基础结构

常规页面结构建议：

```text
顶部状态栏 / 导航栏
↓
搜索栏或页面核心入口
↓
白色卡片模块
↓
列表 / 状态 / 内容区
↓
底部安全区或底部操作区
```

页面整体以纵向布局为主，避免复杂栅格。卡片宽度常用 `351px`，即在 `375px` 画板中左右各保留 `12px`。

### 3.4 自动布局要求

AI 生成页面必须符合移动端自动布局思路，页面结构应便于后续在 Figma 中整理为 Auto Layout。

#### 自动布局原则

- 页面整体采用从上到下的纵向布局，不要使用杂乱的自由摆放。
- 页面根容器需要限制在目标画板 / 视口高度内，不要因为内容溢出产生浏览器滚动条或画板外内容。
- 同一模块内的标题、说明、列表、按钮需要保持清晰的父子层级。
- 卡片、列表项、按钮、标签等组件需要使用统一的间距、圆角、字号和对齐方式。
- 模块之间的垂直间距应稳定，建议控制在 `10–14px`。
- 页面左右边距保持一致，不要出现一边贴边、一边留白过大的情况。
- 列表项内部建议采用横向自动布局：左侧主信息，右侧状态、箭头或操作入口。
- 按钮、搜索框、卡片等组件需要具备可复用结构，不要生成无法拆解的整块图片式 UI。
- 底部操作区需要固定在页面底部安全区上方，并与页面主体内容形成明确层级。
- 列表内容较多时，优先使用卡片内固定高度列表、分页、筛选或更多入口；不要让整个页面出现明显滚动条。

#### 避免的布局问题

- 不要让文字、图标、按钮出现随意漂浮。
- 不要让卡片间距忽大忽小。
- 不要让同类列表项高度不一致。
- 不要让组件左右对齐混乱。
- 不要为了装饰破坏页面栅格、边距和阅读顺序。

## 4. 页面基础组件规范

### 4.1 顶部导航栏

顶部导航风格接近 iOS 小程序导航栏。

#### 结构特征

- 页面宽度：375px
- 顶部导航总高度约：93px
- 状态栏高度约：51px
- 导航标题位于中间，字号约 15–16px，Medium，黑色。
- 左侧使用关闭 / 返回图标。
- 右侧使用更多图标。
- 导航底部可以有 1px 极淡分割线。
- 导航背景多数为白色，也可以与页面浅绿色渐变氛围融合。

#### 视觉要求

- 导航栏必须轻，不要做大面积深色背景。
- 标题居中，图标左右对称。
- 图标使用线性 / 极简样式，颜色为黑色或深灰。
- 不要使用厚重投影或复杂装饰。

---

### 4.2 背景与氛围层

#### 常规背景

默认页面背景使用：

```css
background: #F7F7F7;
```

#### 顶部氛围渐变

在城市选择、搜索类页面中，可以使用顶部浅绿色氛围渐变：

```css
background:
linear-gradient(180deg, rgba(211, 255, 246, 0.9) 3%, rgba(245, 245, 245, 0) 46%),
#F7F7F7;
```

#### 使用原则

- 渐变只作为轻微氛围，不要喧宾夺主。
- 页面主体仍以白色卡片和浅灰背景为主。
- 不要使用深色科技背景、复杂光效、强霓虹或大面积玻璃拟态。

---

### 4.3 卡片

卡片是本项目主要信息承载容器。

#### 基础卡片样式

```css
background: #FFFFFF;
border-radius: 8px;
padding: 14px 12px;
box-shadow: none;
```

#### 常用卡片尺寸

- 宽度：351px
- 左右边距：12px
- 圆角：8px
- 内部间距：10–14px

#### 卡片内容结构

```text
模块标题 16px Medium #333
↓
正文 / 列表 / 标签 / 操作入口
```

#### 视觉要求

- 卡片之间保持 10px 左右间距。
- 卡片不使用明显投影。
- 通过背景色差、圆角和留白区分模块。
- 同一页面的卡片圆角、边距、标题样式要统一。

---

### 4.4 搜索框

搜索框用于城市搜索、服务搜索等输入场景。

#### 推荐样式

```css
height: 32px;
background: #FFFFFF;
border-radius: 40px;
padding: 7px 10px;
font-size: 12px;
color: #999999;
```

#### 搜索框结构

```text
搜索图标 12px
占位文字 12px #999999
```

#### 视觉要求

- 搜索框为胶囊圆角。
- 图标使用线性风格，大小约 12px。
- 占位文字居中或左偏居中，但整体不要显得拥挤。
- 搜索框外部上下留白 7–10px。

---

### 4.5 列表

城市列表、搜索结果、服务列表等都应保持清爽的列表风格。

#### 列表项样式

```css
font-size: 14px;
font-weight: regular;
color: #333333;
border-bottom: 1px solid rgba(0,0,0,0.04);
padding-bottom: 10px;
```

#### 列表层级

- 字母分组：`#A9A9A9`，14px Medium。
- 城市名称 / 主内容：`#333333`，14px Regular。
- 状态标签：绿色文字 + 浅绿色背景。
- 筛选入口：12px，`#999999`。

#### 列表设计要求

- 列表行之间保持均匀节奏。
- 分割线极淡，不要用深灰线。
- 行内容左对齐，状态标签右对齐。
- 当有右侧索引字母时，字母索引应靠右，字号 12px 左右，颜色弱化。

---

### 4.6 标签

#### 已开通标签

```css
background: rgba(199, 246, 235, 0.20);
color: #1DC2A5;
border-radius: 4px;
padding: 2px 6px;
font-size: 11px;
font-weight: Medium;
```

#### 标签使用原则

- 标签只用于状态说明，不要过多堆叠。
- 标签面积要小，不能抢主内容视觉。
- 状态文字尽量简短，例如「已开通」「已完成」「待确认」。

---

### 4.7 按钮

#### 主按钮

主按钮用于最重要的页面动作，例如「查看其他城市」「立即投票」「确认提交」。

```css
height: 44px;
border-radius: 82px;
background: linear-gradient(133deg, #24D2A3 5%, #22BEA3 106%);
color: #FFFFFF;
font-size: 14px;
font-weight: Bold;
```

#### 主按钮布局

- 常用宽度：335px
- 左右边距：20px
- 圆角大，接近胶囊按钮。
- 文案居中。
- 不要使用强投影。

#### 次级按钮

次级按钮可以使用白底、浅灰底或弱边框样式，文字使用 `#333333` 或 `#777777`。

#### 操作文字

例如「重新定位」这类轻量操作文字：

```css
font-size: 12px;
font-weight: Medium;
color: #1DC2A5;
```

---

### 4.8 底部操作面板

投票、确认、提交成功等页面可使用底部面板承载关键操作。

#### 底部面板特征

- 面板贴近底部。
- 宽度 375px。
- 背景以白色为主。
- 顶部可以有浅绿色到白色的柔和渐变。
- 内容区左右 padding 20px。
- 按钮放在面板下方，宽度 335px，高度 44px。
- 保留 iPhone Home Indicator 底部安全区。

#### 面板视觉要求

- 面板不要像后台弹窗，要更像移动端底部内容区。
- 上方可以有轻微异形或渐变，但不要太复杂。
- 重点标题、数据、按钮形成清晰阅读顺序。

---

### 4.9 数据模块

用于投票人数、订单数量、金额等重点数据展示。

#### 样式建议

```css
background: #F7F7F7;
border-radius: 8px;
padding: 30px 0;
```

#### 数字样式

```css
font-family: DINPro Bold;
font-size: 36px;
font-weight: Bold;
color: linear-gradient(180deg, #24D2A3, #22BEA3);
letter-spacing: -1px to -2px;
```

#### 文本样式

```css
font-size: 13px;
font-weight: Medium;
color: #333333;
```

#### 视觉要求

- 数字是视觉焦点。
- 可适当加入很轻的装饰符号，例如叶片、光点、底部微阴影，但必须克制。
- 数据模块背景不要纯白，建议用 `#F7F7F7` 与白色底部面板区分。

---

### 4.10 空状态 / 未开通状态

城市未开通、暂无数据、无搜索结果等页面应使用轻量插画 + 简短说明。

#### 视觉特征

- 插画居中。
- 插画使用低饱和浅灰、浅绿、白色为主。
- 图形可以是地图、定位点、盒子、任务、票据等服务相关意象。
- 文案颜色使用 `#999999`。
- 插画不要太大，保持页面呼吸感。

#### 推荐结构

```text
顶部导航
↓
中部浅色空状态插画
↓
说明文字 13px #999999
↓
底部操作面板 / 引导按钮
```

---

### 4.11 弹窗

投票确认、重要提醒等可使用居中弹窗。

#### 弹窗样式

```css
width: 300px;
background: #FFFFFF;
border-radius: 8px;
```

#### 蒙层

```css
background: rgba(0,0,0,0.70);
```

#### 弹窗内容

- 标题：16px Medium，`#333333`，居中或左对齐。
- 正文：13px Regular，`#777777`，行高舒适。
- 底部按钮高度约 50px。
- 双按钮时左右均分，中间使用浅灰分割线。
- 确认按钮使用品牌绿 `#1DC2A5`。

---

## 5. 图标与插画规范

### 5.1 图标风格

- 线性图标为主。
- 图标尺寸常用 12px、14px、16px、20px、22px。
- 图标颜色以 `#333333`、`#999999`、`#1DC2A5` 为主。
- 图标应圆润、轻量、现代，不要复杂写实。
- 图标必须是正常可识别的功能图标，例如返回、关闭、搜索、定位、更多、勾选、提示、箭头等。
- 不要使用 Emoji 表情作为图标、按钮图标、状态图标或装饰元素。
- 不要用「一个方框里面放一个字」来代替图标，例如不要用带边框的「搜」「定」「返」「选」「关」作为搜索、定位、返回、选择、关闭图标。
- 不要用乱码、字母、单字、符号块、占位框伪装成功能图标。
- 图标与文字之间需要保持稳定间距，建议图标与文字间距为 `4–6px`。

### 5.2 插画风格

- 轻量扁平 / 轻 3D 均可，但要保持清爽。
- 色彩以浅灰、白色、浅绿为主。
- 插画用于空状态或成功状态，不要抢页面信息。
- 成功状态可用浅绿色徽章、勾选、星光、小装饰，但必须克制。
- 插画中不使用夸张表情包、Emoji 或卡通贴纸风格元素。

## 7. AI 生成页面通用约束

文生图 AI / UI 生成 AI 输出页面时，必须遵守以下要求：

1. 必须生成 `375×812px` 竖版手机界面。
2. 必须是移动端 App / 小程序 UI，不要生成网页端、后台端、平板端或横版界面。
3. 页面内容必须完整适配目标视口，不要出现页面级滚动条或画板外溢出内容。
4. 整体背景使用浅灰 `#F7F7F7`，模块使用白色卡片。
5. 品牌主色以绿色为主，优先使用 `#1DC2A5` 和 `#21B88F`。
6. 主按钮使用 `#24D2A3 → #22BEA3` 绿色渐变。
7. 不要使用强烈投影、复杂纹理、暗黑科技背景、玻璃拟态大面积效果。
8. 页面排版要清爽，留白充足，模块层级清晰。
9. 中文文字要尽量清晰、短句化，不要生成大段密集文案。
10. 组件风格统一：圆角、间距、字号、图标线性风格保持一致。
11. 页面可以有轻微浅绿色氛围渐变，但不要影响内容可读性。
12. 页面中不要出现 Emoji 表情，包括标题、按钮、标签、空状态、成功状态、图标和装饰元素。
13. 图标必须是正常 UI 图标，不要使用文字框、单字框、字母框或符号块代替图标。
14. 页面结构必须符合自动布局逻辑，模块之间、组件内部、列表项之间要有稳定的间距和对齐关系。
15. 生成结果应便于后续在 Figma 中拆分为导航栏、搜索框、卡片、列表项、按钮、标签、底部操作区等独立组件。

## 8. 禁止事项

- 不要生成后台管理系统界面。
- 不要生成网页大屏、PC 官网、数据看板或平板界面。
- 不要在开发页面或生成页面中出现页面级滚动条。
- 不要使用深蓝科技风、霓虹光效、赛博朋克、强 3D 大场景。
- 不要大面积使用黑色或深色背景。
- 不要使用复杂阴影、厚重描边、玻璃拟态大面积卡片。
- 不要把品牌绿色用于所有文字，否则会失去层级。
- 不要让卡片太密集，必须保留移动端呼吸感。
- 不要让按钮颜色偏蓝、偏紫、偏红，主操作必须保持绿色系。
- 不要让插画过度写实或过度复杂，插画应轻量、淡雅、辅助表达状态。
- 不要生成无法识别的文字堆叠，文案应短、清晰、结构化。
- 不要出现 Emoji 表情符号。
- 不要使用「方框加单字」的形式冒充图标。
- 不要生成不可拆解、不可复用、无法整理成自动布局的页面结构。
- 不要让图标、文字、按钮随意漂浮或脱离组件容器。

## 10. 一句话设计方向总结

这是一个以绿色为品牌识别、以白色卡片和浅灰背景为基础、强调轻量清爽和服务亲和感的 C 端到家服务类移动端 UI 设计体系；生成任何页面时，都应优先保证移动端比例、清晰层级、柔和绿色强调、简洁卡片结构、正常图标表达和稳定统一的自动布局节奏。
